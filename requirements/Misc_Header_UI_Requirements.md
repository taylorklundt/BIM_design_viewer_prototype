## Sub-Feature Requirements

### 1. New Header UI (High)

**Summary:** Complete overhaul of the main application navigation header — new layout, branding, and navigation structure to improve usability and accommodate the growing feature set.

**Key Requirements:**

- Redesigned top navigation bar with updated branding and layout
- Clear navigation paths to all major feature areas (2D Overlays, Model Views, Federation, Searchsets, Permissions, etc.)
- Responsive layout that works across desktop screen sizes
- User account menu (profile, settings, logout)
- Project context indicator (current project name, switch projects)
- Notification area for system messages and collaboration updates
- Consistent with the mobile app's visual language where applicable
- Accessible design (keyboard navigation, screen reader compatibility, sufficient contrast)

**Acceptance Criteria:**

- All major feature areas are reachable from the header within 2 clicks
- Current project context is always visible
- Header renders correctly at standard desktop resolutions (1280px to 4K)
- No feature areas are lost or harder to reach compared to the current header
