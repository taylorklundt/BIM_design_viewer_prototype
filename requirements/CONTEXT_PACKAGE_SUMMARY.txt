KEY CONCEPTS EXPLAINED
================================================================================

The 3D Model Viewer is built on these core architecture patterns:

1. EVENT-DRIVEN COMMUNICATION
   - Feature modules emit domain-specific events
   - UI components listen and respond to events
   - ModelViewer aggregates and re-emits events
   - Application handles final events

2. MODULAR FEATURE ARCHITECTURE
   - Core: SceneManager, IFCLoader, ModelViewer
   - Features: Navigation, Selection, Visibility, etc. (isolated)
   - UI: Toolbar, Panels, Menus (listen to features)
   - All features work independently and together

3. STATE AGGREGATION
   - Each feature maintains its own state
   - Features expose getState() / setState() methods
   - ModelViewer.getState() combines all feature states
   - States can be saved to localStorage or exported as JSON

4. THREE.JS + IFC.JS INTEGRATION
   - Three.js provides 3D rendering
   - IFC.js provides BIM model parsing
   - Custom integration layer needed for seamless operation

CRITICAL IMPLEMENTATION DETAILS
================================================================================

GOTCHA #1: InstancedMesh Handling (Files: selection.md, sectionplane.md)
  IFC.js uses InstancedMesh for geometry optimization. When calculating
  world-space coordinates (for picking, normals, etc.):

  WRONG:  Just use mesh.matrixWorld
  RIGHT:  Combine mesh.matrixWorld with instance-specific matrix

  const instanceMatrix = new THREE.Matrix4();
  mesh.getMatrixAt(instanceId, instanceMatrix);
  const worldMatrix = new THREE.Matrix4()
    .multiplyMatrices(mesh.matrixWorld, instanceMatrix);

GOTCHA #2: Clipping Plane Direction (File: sectionplane.md)
  Three.js clips geometry on the POSITIVE side of plane normal.
  To clip "behind" a face, you must NEGATE the normal.

  WRONG:  const clipPlane = new THREE.Plane(normal, constant);
  RIGHT:  const clipPlane = new THREE.Plane(normal.negate(), constant);

GOTCHA #3: Material Cloning (Files: selection.md, visibility.md)
  Always clone materials before modifying to avoid affecting shared materials.

  WRONG:  mesh.material.opacity = 0.5;
  RIGHT:  mesh.material = mesh.material.clone();
          mesh.material.opacity = 0.5;

GOTCHA #4: Element ID Stability (Files: selection.md, viewstates.md)
  IFC express IDs are stable across sessions (good for persistence).
  Three.js UUIDs are unique but not persistent.

  For view state persistence: Use IFC express IDs, not UUIDs

GOTCHA #5: Visibility State with Mixed (File: objecttree.md)
  When a tree node has some children visible and some hidden,
  its visibility state should be "mixed" (not "visible" or "hidden").
  This requires recursive calculation.
